# Daily Report 歸檔規則

`docs/daily_report.md` 預設只保留**最近 7 天**的工作紀錄（滾動視窗），更早的內容歸檔至本目錄，避免主檔案不斷膨脹。

## 歸檔規則

1. **檔名格式**：`YYYY-MM.md`（每月一檔）
   - 例：`2026-04.md`、`2026-05.md`
2. **內容組織**：依日期倒序排列，每天一節 `## YYYY-MM-DD`
3. **觸發時機**：
   - `daily_report.md` 中超過 7 天的內容於下次工作流程啟動時搬入對應月份檔
   - 若該月檔尚不存在則自動建立
4. **副作用**：
   - 主 `daily_report.md` 永遠維持 7 天上限的內容（≤ 50 KB）
   - 歷史完整性透過月份檔保留，git history 仍是最終真相

## 為何不直接 git history？

- git log 翻找特定日期費時，且 `git show` 對於跨多個 commit 的內容拼接困難
- 月份檔可一次瀏覽該月完整脈絡（issue #、PR、實作細節、commit hash）
- email 通知或外部分享時較易引用單一連結

## 與 `docs/changelog.md` 的分工

| 項目 | daily_report / archive | changelog |
|------|------------------------|-----------|
| 內容 | 每日工作流程（commit、issue、lint、模組狀態） | 物理模型 PR 詳述（每個 PR 一節） |
| 頻率 | 每個工作日（可能多日無更新） | 每個物理模型 PR 完成後 |
| 排列 | 月份倒序，月內日期倒序 | PR 編號倒序 |
| 修改性 | 月份檔可校稿，但不重寫 | 既有條目永不修改（accumulate-only）|

## 操作備忘

- 主 `daily_report.md` 中超過 7 天的章節 → 剪入對應 `YYYY-MM.md`
- 歸檔時保留章節層級（`## 2026-04-30` → `## 2026-04-30`）以利快速跳轉
- 不需要在歸檔檔中加入 ToC 或交叉連結，每個檔案獨立完整
