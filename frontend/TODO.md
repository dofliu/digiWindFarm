# 專案待辦事項與開發藍圖

本文件追蹤「智慧型風場監控與維護系統」的開發藍圖。

## 第一階段：核心功能整合與強化

- [ ] **歷史數據查詢頁面**:
    - [ ] 建立一個用於查詢歷史風機數據的新視圖。
    - [ ] 允許使用者選擇風機和日期範圍。
    - [ ] 以圖表和表格兩種格式顯示數據。
- [ ] **強化維修報告**:
    - [ ] 在工單詳細資料彈出視窗中新增「標準作業程序（SOP）檢查清單」。
    - [ ] 新增一個用於記錄「已消耗零件/物料」的區塊。
- [ ] **表單草稿自動儲存**:
    - [ ] 在工單詳細資料彈出視窗中，自動將未儲存的筆記和照片儲存到 LocalStorage，以防止因意外關閉/重新整理而導致的資料遺失。
- [ ] **搜尋與篩選**:
    - [ ] 在維護中心的工單列表中實作搜尋和篩選功能（依風機、技師、狀態、日期）。

## 第二階段：後端整合與離線支援

- [ ] **API 整合**:
    - [ ] 將模擬數據的 hooks (`useMockTurbineData`, `useMockMaintenanceData`) 替換為對真實後端 API 的 fetch 呼叫。
- [ ] **使用者驗證**:
    - [ ] 使用 JWT 或類似機制實作登入/登出流程。
- [ ] **雲端圖片儲存**:
    - [ ] 與雲端儲存服務（如 S3、Google Cloud Storage）整合，用於上傳工單照片。
- [ ] **離線支援**:
    - [ ] 使用 Service Worker 快取應用程式資產。
    - [ ] 使用 IndexedDB 儲存工單數據，允許技師在離線時查看和更新。
    - [ ] 實作背景同步，以便在恢復連線時將變更推送到伺服器。

## 第三階段：主管與管理功能

- [ ] **管理儀表板**:
    - [ ] 為管理人員開發一個新的儀表板，以查看關鍵效能指標（KPIs），如可用率、平均修復時間（MTTR）和故障類型分佈。
- [ ] **報表生成**:
    - [ ] 新增將維護報告和營運數據匯出為 PDF 或 CSV 檔案的功能。
- [ ] **工單審核流程**:
    - [ ] 實作一個功能，讓主管可以審核、批准或拒絕已完成的工單。
- [ ] **角色權限管理 (RBAC)**:
    - [ ] （需要後端支援）建立不同的使用者角色（操作員、技師、主管），並賦予不同的權限。

## 第四階段：進階 AI 整合

- [ ] **智慧文字建議**:
    - [ ] 使用 Gemini 根據故障描述和歷史數據，為維護筆記建議標準化文字。
- [ ] **維修照片分析**:
    - [ ] 使用視覺模型（如 Gemini Vision Pro）分析上傳的照片，以識別零件損壞或根據 SOP 驗證維修品質。
- [ ] **語音轉文字輸入**:
    - [ ] 整合 Web Speech API，允許技師口述維護筆記。
- [ ] **預測性維護**:
    - [ ] 利用 Gemini 對歷史數據的分析能力，主動建議未來可能發生的故障。
